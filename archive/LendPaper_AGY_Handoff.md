## Overview

LendPaper is a SaaS platform that allows lenders, Independent Sales Organizations (ISOs), and brokers to generate financial estimate documents (PDFs) for their clients. This handoff covers four build areas:

1. PDF footer/attribution copy (tiered by plan)
2. In-document legal disclaimer (micro-copy, on every generated PDF)
3. Hosted legal disclaimer page (`lendpaper.com/legal/estimates`)
4. Favicon strategy for generated documents vs. main site

All four areas are **required** before the product goes to any external users.

## Section 1 — PDF Footer: "Powered By" + PLG Copy

### Background
The PDF footer serves a dual audience simultaneously:
- **End-clients** (borrowers) should read past it — it's attribution, not content for them.
- **Brokers/ISOs/Lenders** (the paying users) should feel subtle friction or a clear upgrade prompt depending on their tier.

### Implementation Logic
Footer copy is **rendered conditionally by plan tier** in the PDF generation template. Three variants:

---

#### Tier: FREE
```
⚙ Built with LendPaper Free  ·  lendpaper.com
Brokers: Remove branding, add your logo, and unlock custom calculators → lendpaper.com/upgrade
```
**Notes:**
- "Free" label is intentional — it creates visible status differentiation that motivates upgrade.
- The `→ lendpaper.com/upgrade` should be a live hyperlink in digital PDF output.
- Font: 8pt, color #6B7280 (muted gray). Do not make this visually prominent to end-clients, but keep it legible.

---

#### Tier: PRO / BRANDED
```
Generated via LendPaper  ·  lendpaper.com
Remove this footer and add your logo — ask about white-label plans.
```
**Notes:**
- Softer tone. Pro users are already paying; this is a gentle upsell to white-label.
- Same typography treatment as Free tier.

---

#### Tier: WHITE-LABEL
```
Powered by LendPaper  |  info@lendpaper.com  |  lendpaper.com
```
**Notes:**
- Minimal attribution only. No upsell copy.
- If the Enterprise toggle is enabled for this account, this line is removed entirely and replaced with the broker's own footer string (broker configures this in their account settings — out of scope for this build phase, placeholder only).

---

#### On Every Tier — Legal Micro-Copy (Required)
This line appears **below** the tier-specific footer on every generated document, on every page:

```
ESTIMATES ONLY — NOT FINANCIAL ADVICE. These figures are preliminary estimates generated by the 
user (lender, broker, or ISO) using LendPaper software. LendPaper is a software provider only — 
not a lender, broker, financial advisor, or institution. LendPaper does not verify inputs, 
guarantee calculations, or determine final loan terms. Actual terms are subject to lender 
underwriting and final approval. Full terms: lendpaper.com/legal/estimates
```

**Notes:**
- Font: 7pt, color #9CA3AF (lighter gray than the PLG footer above).
- Must appear on **every page** of multi-page PDFs, not just page 1.
- `lendpaper.com/legal/estimates` must be a live hyperlink in digital output.
- Do **not** conditionally suppress this block on any tier, including White-Label. It is always present.

## Section 2 — PDF Header / Logo Placement

### Layout: Two-Column Header Band
All generated PDFs use a two-column header band separated from document content by a hairline rule.

```
[LEFT COLUMN]                          [RIGHT COLUMN]
Broker/Lender Logo (if Pro+)           □ lendpaper.  (mark + wordmark)
or blank (Free tier)                   · Scenario Modeling Engine
```

**Hairline rule:** 0.5pt, color #E5E7EB, full width, immediately below header band.

**Document title** (e.g., "PAYMENT BREAKDOWN") begins below the rule, centered.

### Tier Behavior

| Tier | Left Column | Right Column |
|---|---|---|
| Free | Empty | Full LendPaper mark + wordmark |
| Pro/Branded | Broker logo (uploaded in account settings) | LendPaper mark, smaller, subordinate |
| White-Label | Broker logo, full weight | LendPaper mark removed OR micro "Powered by LP" badge (founder to decide) |
| Enterprise | Broker logo | Fully custom — founder provides per-account config |

### Logo Specs
- LendPaper mark: use existing `□ lendpaper.` SVG asset from homepage.
- Right column mark height: 28px at standard PDF resolution.
- Broker logo upload: support PNG and SVG, max 400px wide, 80px tall, with white or transparent background. Constrain on render.

## Section 4 — Favicon Strategy

### Two States: Product Mode vs. Artifact Mode

The LendPaper favicon should have two distinct states that share the same visual DNA but signal different contexts.

---

### State A — Product Mode (Main Site / App)
Use existing LendPaper mark as-is. No changes needed.

---

### State B — Artifact Mode (Generated PDF browser tab / document identity)

Apply the following modifications to the base mark:

**Shape:** Same document-fold glyph (□ with corner fold) as the main mark.
- Add a small outbound arrow (`↗`) overlaid in the bottom-right quadrant of the glyph at 32×32 and larger sizes.
- At 16×16, drop the arrow overlay — too much noise. Use the fold glyph alone.
- Alternatively (if arrow reads poorly at 32×32): use the **filled** version of the glyph instead of the outlined version used on the marketing site. This alone signals "different state."

**Color:**
- Main site primary green (current brand value) → shift to a slightly lighter, desaturated slate-green for Artifact Mode.
- Example shift: if main is `#1A3C2E`, Artifact Mode uses approximately `#2D6A4F`.
- Do **not** go to full grayscale — that reads as "disabled." A hue shift within the same green family reads as "same brand, different context."

**Optional Badge:**
- At 32×32 and above: add a 3–4px circular badge in amber/gold (`#D4A017`) in the top-right corner of the favicon canvas.
- This trains users over time: gold badge = LendPaper estimate document. Works similarly to how notification badges signal state in mobile UI.

**Deliverables needed from designer:**
- `favicon-artifact-16.png`
- `favicon-artifact-32.png`
- `favicon-artifact-180.png` (Apple touch icon, for when PDFs are saved/shared on iOS)
- SVG source file for all sizes

## Section 5 — Render Gate (Recommended, Not Optional)

**Problem identified in current build:** The DSCR tool is currently exportable in its empty/zero state. A broker could accidentally hand a client a blank DSCR PDF, which is both a trust problem and exposes LendPaper to liability the disclaimer is not designed to cover (because the document contains no actual content to disclaim).

**Required behavior:**
- Before generating any PDF, run a validation check on key output fields.
- If required output fields are null, zero, or contain only default/placeholder values: **block the export**.
- Surface a UI message: *"Complete your inputs to generate this document."*
- Do not allow download of any blank or near-blank Generated Document.

**Fields to gate by document type:**

| Document Type | Required Non-Zero Fields Before Export |
|---|---|
| Payment Breakdown / Amortization | Financing amount, term, cost per dollar borrowed |
| DSCR Analysis | At least one EBIDA field + one deal scenario amount |
| Fundability / Stacking | Revenue input, at least one position modeled |
| SBA Fees | Loan amount, term, rate |
