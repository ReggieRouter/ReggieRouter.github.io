Help & FAQ

What's the difference between Name and Display Name?
Name is the canonical name from the scrape (source of truth for matching). Display Name is what shows on the waterfall. Leave it blank to use the full name.
How do I manually override a scraped value?
Go to Edit All Fields, change the value, and save. Manually edited fields are marked with 🔒 — future scrapes won't overwrite them automatically.
How do I unlock a manually overridden field?
In Edit All Fields, the field shows a 🔒 icon. Click Unlock next to it to allow scrapes to update it again.
How do I add a new data field to all lenders?
Go to supabase.com → your project → Table Editor → lender_data → Add Column. The admin dashboard picks it up automatically after you add it here.
What happens when I click Publish?
You'll see a command to run in your terminal:
python3 ~/lendpaper-engine/publish.py
This reads all approved lenders from Supabase, regenerates the lenderData block in waterfall.html, commits to GitHub, and Netlify deploys within ~60 seconds.
Why do I have to review at least one lender before publishing?
Spot-checking prevents bulk scrape errors from going live. You only need to open and review one lender per session — the system tracks it automatically.
Where do I change a lender's logo?
Drop the file in public/assets/lenders/, then update the Logo URL field in Edit All Fields.
How often do scrapes run?
Weekly on Sundays at midnight (if the scheduler is running). Run python3 ~/lendpaper-engine/scheduler.py to start it. A proper launchd cron job is planned.
How does the spot check before publishing work?
Before each publish, you'll be shown a random sample of recently-changed lenders and asked to visit their live website. The sample size uses a statistical formula (√N, min 3, max 8) giving ~95% confidence in data accuracy. Click "Visit →" for each one, then proceed to publish.
How do I train the scraper to avoid future data misses?
Each lender has a Scraper Notes field in the Live Preview tab. Use it to leave notes like: "Min FICO is on the /qualifications page, not the homepage" or "Rate shown is factor rate, not APR." These notes are stored in localStorage and exported via Export Scraper Notes in the Dashboard. Share the exported JSON with the scraper developer or drop it in ~/lendpaper-engine/hints/ so the scraper reads them before each run.

Active Publish Rules

Pending Changes Threshold
If more than 5 changes are unreviewed, publishing is blocked. Work through the queue first.
Required Fields Gate
Before publishing, all approved lenders must have min_fico, min_tib_months, min_monthly_revenue, and product_types filled. You'll be warned with a list of offenders.
Stale Lender Warning
If more than 20% of approved lenders haven't been updated in 90+ days, you'll see a warning before publishing. Not a hard block.
Spot Check (Mandatory)
Before each publish you must visit a random sample of lender sites (√N formula, min 3, max 8) to verify live accuracy.
Suspicious Zero / Blank Changes
Any scrape that sets a numeric field to 0 or empties a filled array is flagged ⚠ in the queue and sorted to the bottom. Approving one requires manual confirmation.
Source URL Required (Soft)
Approving a change for a lender with no source URL triggers a confirmation. Prevents approving data you can't verify against a live site.
Lock on Reject
After rejecting a scraped value, a toast offers to lock that field. Locked fields are skipped by future scrapes.
